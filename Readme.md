# Painters' Diary

A social platform for artists, creatives, and culture lovers to share work, discover inspiration, and connect with communities.

## Key features
- Create artist profiles and galleries
- Post images, sketches, process videos, and written notes
- Follow artists, like, comment, and curate collections
- Tagging, search, and discovery by medium, style, or theme
- Events, exhibitions, and collaboration boards
- Privacy controls and content moderation tools
- Responsive web and mobile-friendly design
- Export portfolio and printable artist statements

## Tech stack (suggested)
- Frontend: React / Next.js, Tailwind CSS
- Backend: Node.js / Express or Fastify
- Database: PostgreSQL (or MongoDB for flexible schema)
- File storage: S3-compatible storage
- Auth: OAuth2 / JWT
- Search: ElasticSearch or PostgreSQL full-text
- CI/CD: GitHub Actions, Docker

## Quick start
1. Clone the repo
```
git clone https://github.com/your-org/painters-diary.git
cd painters-diary
```
2. Install dependencies
```
npm install
```
3. Copy and configure environment
```
cp .env.example .env
# set DATABASE_URL, S3 credentials, AUTH keys, etc.
```
4. Run locally
```
npm run dev
```

## Architecture overview
- Client communicates via REST/GraphQL to backend
- Media uploaded to object storage; thumbnails and derivatives generated by worker queue
- Background jobs handle search indexing, image processing, and notifications
- Role-based access for moderation and content review

## Content and community guidelines
- Respect creators and intellectual property
- No harassment, hate speech, or illegal content
- Properly credit references, collaborators, and sources
- Use moderation tools and reporting for violations

## Privacy & Safety
- Offer user-controlled visibility (public, followers-only, private)
- Rate limits, spam detection, and manual moderation queues
- GDPR/CCPA compliance guidance for deployments
